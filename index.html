Can You Match π?

Choose an integer \(N_{\max}\) and a real \(\lambda\), then compute the truncated series and see how many digits match the true value of \(\pi\).
Formula
$$ f_{\pi} = 4+\sum_{n=0}^{\infty} \frac{1}{n!} \left(\frac{1}{\lambda +n}-\frac{4}{2 n+1}\right) \left(\frac{\left(2n+1\right)^{2}}{4 (n+\lambda )}-n\right)_{n-1} $$

The graph plots \[ -\log_{10}\bigl(|f_{\pi}(N,\lambda)-\pi|\bigr) \] against \(N\). A larger value means a smaller error, so higher points correspond to better agreement with the exact value of \(\pi\).
Algorithm used for the calculation
1. Fix the values of \(N_{\max}\) and \(\lambda\).

2. For each integer \(n=1,2,\dots,N_{\max}\), compute the term \[ \frac{1}{n!} \left(\frac{1}{\lambda+n}-\frac{4}{2n+1}\right) \left(\frac{(2n+1)^2}{4(n+\lambda)}-n\right)_{n-1}. \]
3. The rising Pochhammer symbol is evaluated as the finite product \[ (a)_{n-1}=a(a+1)(a+2)\cdots(a+n-2), \qquad (a)_0=1. \]
4. Add all these terms and then add the leading \(4\). This gives the approximation \(f_{\pi}\).

5. Write \(f_{\pi}\) to 100 decimal places and compare it with the stored exact value of \(\pi\), digit by digit after the decimal point, until the first mismatch appears.
How the matching works
The exact value of \(\pi\) is stored up to 100 decimal places. After computing \(f_{\pi}\), the website compares the decimal digits from left to right after the decimal point.

The number of matching digits is the number of consecutive digits that agree before the first mismatch.

In the displayed approximate value, all matched digits are shown in green and the first mismatched digit is shown in red. The remaining digits are shown in a lighter color.
